## What was added

### Rust backend (`src-tauri/src/lib.rs`)

- `PullRequestWithChecks { pr: Option<GitHubPullRequest>, checks: Vec<GitHubCheckRun> }` — the bundle returned by the new IPC. `pr` is `None` when the branch has no associated PR (success case, not an error).
- `gh_pr_view_blocking(cwd, branch)` — runs `gh pr view <branch> --json number,title,body,state,headRefName,baseRefName,url,isDraft`, returns the parsed PR. Detects "no PR for branch" via stderr message matching (tolerant of future gh versions instead of exit-code pinning). DTO maps `headRefName → head_ref`, `baseRefName → base_ref`, `isDraft → draft`, `url → html_url` to preserve snake_case on the wire.
- `gh_pr_checks_list_blocking(cwd, branch)` — runs `gh pr checks <branch> --json name,state,conclusion,detailsUrl,startedAt,completedAt`, returns `Vec<GitHubCheckRun>`. Empty stdout → empty list (no escalation — older gh versions without `--json` for `pr checks` don't break the UX). Auth / rate-limit errors still bubble up via `classify_gh_error`. DTO maps `detailsUrl → html_url`, `startedAt → started_at`, `completedAt → completed_at`. The synthetic `id` is the position in the array (gh pr checks doesn't expose the GitHub check-run id; the URL is what the UI uses).
- `classify_gh_error(status, stderr) -> anyhow::Error` — returns `Err("<code>: <message>")` with stable code prefixes the UI matches on:
  - `gh_unavailable` — `gh` exited with no output (e.g. binary missing on PATH).
  - `gh_unauthenticated` — stderr contains "not logged in", "auth login", "not authenticated", or "no oauth token".
  - `rate_limited` — stderr contains "rate limit" or "api rate limit exceeded".
  - `gh_error` — fallback for anything else.
- `github_pr_checks_fetch_blocking(project_id, branch) -> Result<PullRequestWithChecks>` — loads the project by id, resolves its `root_path`, runs both `gh` calls. Returns `anyhow::Error` so the spawn_blocking closure can use the same `?` operator as the rest of the file.
- `#[tauri::command] async fn github_pr_checks_fetch(project_id, branch) -> Result<PullRequestWithChecks, String>` — IPC entry. `async` + `tauri::async_runtime::spawn_blocking` per the long-running-IPC discipline in CLAUDE.md (two `gh` subprocesses, each up to a few hundred ms on a cold PATH).
- Wired in `invoke_handler` next to `github_status`.
- Two new unit tests (89 total, all pass):
  - `pr_checks_wire_format_round_trips` — deserializes realistic snake_case JSON for `GitHubPullRequest` + `Vec<GitHubCheckRun>`, verifies every field. Also pins the `pr=null → JSON null` serialization contract (UI distinguishes "no PR" from "never fetched" via the explicit null).
  - `classify_gh_error_emits_stable_codes` — verifies all four error code prefixes plus the success-status branch.

### TS frontend

**`src/lib/types.ts`:** Added `PullRequestWithChecks` interface (snake_case on the wire, mirrors the Rust struct).

**`src/lib/ipc.ts`:** Added `githubPrChecksFetch(projectId, branch)` wrapper. Imports `PullRequestWithChecks` from `./types` (no parallel type declaration in `ipc.ts`, per the file's convention).

**`src/store/app.ts`:**
- New state: `prChecks: Record<wsId, { pr, checks, loading, error, fetchedAt }>`.
- New actions: `setPrChecks(wsId, payload)`, `setPrChecksLoading(wsId, loading)` (separate so the refresh button can flip the flag without overwriting the prior data), `clearPrChecks(wsId)`.
- `setPrChecksLoading` uses the `if (cur?.loading === loading) return s;` no-op return — same referential-identity trick `setWorkState` uses to avoid re-render churn.
- New frozen constant `EMPTY_PR_CHECKS` + new selector `usePrChecks(wsId)` — falls back to the frozen constant when a workspace has no entry (never fetched) so the snapshot stays referentially stable across re-renders. Matches the `EMPTY_TABS` / `EMPTY_DIFF_COMMENTS` pattern at `app.ts:899-900`.

**`src/components/workspace/RightPanel.tsx`:**
- Added `"checks"` to `FootTab` type.
- Added `Checks` tab button to BOTH the single-repo strip (next to Terminal, with a `GitPullRequest` icon) and multi-repo Row 2 (next to Setup/Run, with the same icon). Always present — not gated on `showRunTab` or `spotlightAvailable`.
- Added `ChecksRefreshButton` (right-side of the strip) for both single-repo + multi-repo. Disabled + spinning icon while loading. Has `data-testid="checks-refresh"` per the plan.
- Added `ChecksContent` component (the tab body):
  - Fetches on mount (via `useEffect` with `[ws.id, ws.project_id, ws.branch, data.loading]` deps).
  - Fetches on tab focus: handled implicitly by the unmount-on-tab-switch architecture. The user clicks Setup → Checks, the Checks tab unmounts, then remounts when re-selected, the remount triggers the mount-fetch.
  - Manual refresh: refresh button calls `setPrChecksLoading(wsId, true)`; the effect's dep on `data.loading` re-fires the IPC.
  - Renders the PR header (title, number, draft badge, head→base branches, clickable to open via `openPath`).
  - Renders the check-runs list via `CheckRunRow` — status icon (spinner for in-progress, `CheckCircle2` for success, `XCircle` for failure, `MinusCircle` for skipped/neutral/cancelled, `AlertCircle` for timed_out/action_required/stale), name, conclusion text, "started Xs ago" / "completed Xs ago" via `timeAgo`, hover-revealed `ExternalLink` icon. Clickable to open the run URL via `openPath`.
  - Empty state for no PR: "No PR or checks found for this branch" + branch name in a code element.
  - Error state for `gh_unavailable` / `gh_unauthenticated`: "Install and authenticate gh" + contextual hint (auth vs install) + "Try again" button.
  - Error state for `rate_limited`: "GitHub API rate limit hit" + "Try again" button.
  - Error state for unknown code: AlertCircle icon + the raw error message + "Try again" button.
  - Loading state: spinner (only when `loading && fetchedAt === null`, so refreshes don't blank the existing data).
