New shortcuts
-------------
All four live in the new "GitHub" ShortcutGroup (appended to GROUP_ORDER, so the
ShortcutsHelpDialog + ShortcutsSection settings list both render them under that
header). Each default is ⇧⌘ + a free letter, deliberately picked to dodge the
long-standing ⌘1..9 / ⌘[/] / ⌘W / ⌘L / ⌘T / ⇧⌘[/] workspace-nav set the spec
called out.

  id              default     action
  pr-create       ⇧⌘P         useUI.openPrCreate(wsId) on the active workspace
  issue-import    ⇧⌘I         useUI.openIssueImport(ws.project_id) on the active workspace's project
  open-checks     ⇧⌘K         window.dispatchEvent("termic-open-checks") → RightPanel useEffect listener
                               switches footTab to "checks" + expands the footer; gates on wsId match
                               so a stale event from a previous activation can't redirect focus.
  open-history    ⇧⌘H         useApp.setView("dashboard" | "history") — toggles; on-history jumps back

⇧⌘P is the QA scenario's required binding for PR-create.

All four are rebindable via the existing Settings → Shortcuts recorder (the
SHORTCUT_DEFS array is the single source of truth; the conflict detector in
ShortcutsSection will flag any collision the user creates).

No-op cases (consistent with the existing pattern):
  - pr-create / issue-import / open-checks: return early when there's no active workspace.
  - open-history: always fires (history is global, not workspace-scoped).

lines 256-274: the new useEffect mirrors the existing event-listener pattern
(setup-script-output listeners at lines 98-126, spotlight listeners at
lines 178-220) and only adds a window-level add/remove pair, identical in
shape to the existing ones.

    `e.metaKey=true, e.shiftKey=true, key="p"` against `B("p", {cmd:true, shift:true})`.
